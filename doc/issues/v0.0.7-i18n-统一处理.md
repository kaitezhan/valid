# fail 的处理调整

## 调整为接口

## 限制结果

类似于 java 的文件遍历，限制结果为 FailTypeEnum

这样类比于原来的优点：

（1）用于可以自定义实现方式

（2）所有结果，我们只关心 FailTypeEnum 的值，来处理逻辑。

不影响整体逻辑。

## 可以支持的模式

（1）fail-over

（2）fail-fast

（3）fail-throws

直接抛出异常，中断当前模式

（4）fail-skip

失败了，跳过当前异常的处理。

当前，一般这是不可取的。只是举个例子。
